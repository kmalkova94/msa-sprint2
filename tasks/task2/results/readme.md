## Обзор архитектуры
### Текущее состояние (As-Is)

- Монолит: Spring Boot приложение с REST API (порт 8084)

- Booking Service: gRPC микросервис для бронирований (порт 9090)

- Booking History: Сервис истории бронирований (порт 9091)

- Базы данных: Отдельные PostgreSQL для каждого сервиса

- Мессенджер: Kafka для асинхронной коммуникации

## Промежуточное состояние
Промежуточное состояние описано в hotelio_diagram.puml

## Стратегия миграции данных

- Этап 1. Подготовка (Pre-Migration)

  - Монолит продолжает работать со своей БД
  - Внедряется gRPC-прокси для перенаправления запросов бронирования 
  - Новые бронирования создаются через booking-service
- Этап 2. Переключение (Cut-Over)

  - Поэтапное переключение трафика
  - Валидация данных
- Этап 3. Консолидация (Consolidation)

    - Деактивация старого кода в монолите 
    - Очистка дублирующихся данных 
    - Оптимизация схемы данных в новом сервисе
